# Dino Mutant Bot 使用教學

這份教學適用於 Windows 電腦上的 BlueStacks 與 Dino Mutant。Bot 會透過
ADB 讀取遊戲畫面並操作，不需要搶走滑鼠或鍵盤焦點。

> 請自行確認遊戲規則、帳戶安全與使用風險。本工具不會提供帳號或登入資料。

## 1. 需要準備什麼

1. Windows 10 或 Windows 11。
2. BlueStacks 5，並已安裝、登入 Dino Mutant。
3. Android SDK Platform Tools（提供 `adb.exe`）。
4. 完整的 `DinoMutantBot` 資料夾；請不要只複製其中的 `app` 資料夾。

完整分享版已附帶可攜式 Python，不需要另外安裝。若資料夾未封裝 Python，第一次
雙擊時也會詢問是否自動下載安裝。

## 2. 複製與放置 Bot

將整個資料夾複製到本機磁碟，例如：

```text
D:\DinoMutantBot
```

建議將資料夾直接放在 `D:` 根目錄，不要放在 OneDrive、桌面或路徑含特殊字元的資料夾。

## 3. 設定 BlueStacks

1. 開啟 BlueStacks，進入「設定 → 進階」。
2. 開啟 Android Debug Bridge（ADB）。
3. 確認 ADB 位址為 `127.0.0.1:5555`。
4. 將遊戲畫面設為直向 `900 × 1600`。Bot 的辨識素材以此畫面建立。
5. 啟動 Dino Mutant，並停在主地圖或採集地圖。

BlueStacks 必須保持開啟，但 Bot 不需要 BlueStacks 是目前作用中的視窗。螢幕可以依 Windows 電源設定關閉；電腦本身不可進入睡眠。

## 4. 設定 ADB 路徑

使用記事本開啟：

```text
D:\DinoMutantBot\app\config.json
```

Bot 會依序尋找目前帳號的 Android SDK、環境變數與 BlueStacks 內附的 ADB。一般情況
保持 `"executable": null` 即可；只有自動尋找失敗時才改成完整路徑。例如：

```json
"adb": {
  "executable": "C:\\Users\\你的Windows帳號\\AppData\\Local\\Android\\Sdk\\platform-tools\\adb.exe",
  "serial": "127.0.0.1:5555",
  "connect_on_start": true,
  "timeout": 5.0
}
```

注意：JSON 裡的 Windows 路徑要使用兩個反斜線 `\\`。存檔時請保留 UTF-8 編碼，不要新增逗號或刪除引號。

若電腦沒有 `adb.exe`，請安裝 Android SDK Platform Tools 或 Android Studio 的 Android SDK Platform-Tools 元件，再回到此處設定路徑。

## 5. 第一次啟動

雙擊：

```text
D:\DinoMutantBot\start-bot.cmd
```

啟動器會自動檢查 Python、ADB、辨識素材與畫面擷取。全部通過才會啟動 Bot；若顯示
`FAIL`，先閱讀下方「常見問題」，不需要另外執行檢查指令。

## 6. 啟動與停止

`v0.1.2` 會開啟兩個視窗：

- 繁體中文控制視窗：查詢狩獵與失敗次數、調整參數、重啟及執行診斷。
- LOG 視窗：持續顯示 Bot 原本的詳細執行紀錄。

控制視窗的指令如下：

```text
S  顯示狩獵、信箱、操作、失敗、黑屏與最近動作
T  選擇 fast、safe 或自訂毫秒數，套用後自動重啟
P  切換本機接口 Port；Port 被占用時會顯示程式與 PID
D  開啟診斷工具（環境檢查、ADB 截圖、JSON、原始日誌）
A  顯示 AI 接口網址
R  以目前參數重啟
Q  停止 Bot
```

預設以快速模式啟動，也可開啟 `cmd` 或 PowerShell 預先指定速度：

```bat
D:\DinoMutantBot\start-bot.cmd fast
D:\DinoMutantBot\start-bot.cmd safe
D:\DinoMutantBot\start-bot.cmd fast 8877
```

- `fast`：一般點擊 500 ms、選恐龍 500 ms、狩獵 1500 ms、確認 2000 ms、無目標掃描 250 ms。
- `safe`：選恐龍 1500 ms、狩獵 5000 ms、確認 3000 ms。

一般使用者直接在控制視窗按 `T` 自訂即可。進階使用者也能直接執行（單位為毫秒）：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass `
  -File D:\DinoMutantBot\app\scripts\run-windows.ps1 `
  -Mode runtime -Speed fast `
  -DinosaurDelayMs 800 -HuntButtonDelayMs 2500 `
  -HuntConfirmDelayMs 1800 -IdleDelayMs 150
```

套用新參數時，控制視窗會自動停止並重啟 LOG 視窗。正常情況會依序看到：

```text
Detect
Planning
Action
Verify | Success
```

在控制視窗按 `Q` 即可停止 Bot。完整日誌會保存在：

```text
D:\DinoMutantBot\app\logs\YYYYMMDD.log
```

## 7. AI 與外部工具接口

Bot 啟動後，同一台電腦上的 AI 或工具可直接讀取：

```text
完整狀態  http://127.0.0.1:8765/status
最近動作  http://127.0.0.1:8765/actions
目前參數  http://127.0.0.1:8765/settings
健康檢查  http://127.0.0.1:8765/health
安全停止  POST http://127.0.0.1:8765/control/stop
```

這些接口只監聽本機。狀態接口為唯讀，唯一的 HTTP 控制是安全停止；AI 無法直接點擊
遊戲或送出任意指令。Port 可在控制視窗按 `P` 更改。`/status` 會回傳成功狩獵、
信箱循環、操作總數、驗證失敗、黑屏、遊戲重啟、目前階段與最近操作。

若選定的 Port 已被占用，直接按 Enter 或輸入 `N` 會改用下一個 Port。只有程序命令列、
`/health` API 身分與 PID 都確認為 Dino Bot 時才會出現 `K` 清理選項；輸入確認碼後
系統還會再檢查一次占用者，避免誤關剛好接手 Port 的其他程序。清理必須依畫面輸入
`CLEAN-<Port>`，若需強制結束還要再輸入 `FORCE-<PID>`。未知程式不會出現清理
選項，也不會被控制台關閉。

資料夾內附 Codex Skill。使用 Codex 開啟這個資料夾後可直接輸入：

```text
$control-dino-bot 幫我查狩獵進度
$control-dino-bot 用 8877 Port 查詢狀態
$control-dino-bot 請重新啟動 Bot
```

Skill 只會使用固定控制腳本；啟動、停止與重啟必須由使用者當次明確要求，不會自行
掃描 Port、執行 ADB 指令或探索遊戲畫面。控制腳本會先驗證 `/health`；停止或重啟前
還會交叉確認 API PID、Port 占用者與 Bot 命令列，驗證失敗便不送出控制請求。

## 8. Bot 目前會做什麼

- 辨識安全的恐龍，避開我方藍色路徑與底部 UI。
- 選擇最大群組、按狩獵、按確認，並重新驗證結果。
- 每 10 次重置地圖，讓中央蛋巢回到畫面中心。
- 每 30 次開信箱，依序「全部獲取 → 資源獲取 → 關閉」。
- 隊伍滿額或目標太強時，暫停 5 分鐘後再試。
- 遊戲畫面持續全黑超過 45 秒時，自動重啟遊戲 App；短暫轉場不會重啟。
- 若因其他裝置登入而出現「不同設備的歷史記錄」，會自動選擇此設備並繼續。
- 遊戲重啟後若出現「自動成長結果」，會經由自動戰鬥快捷視窗返回採集地圖。

> 自動選擇此設備代表目前模擬器會接手遊戲連線；若你仍要在另一台裝置操作，請先停止 Bot。

## 9. 常見問題

### 顯示 `ADB executable not found`

`config.json` 的 `adb.executable` 路徑不正確，請確認該位置真的有 `adb.exe`。

### 顯示找不到 ADB 裝置

確認 BlueStacks 已開啟 ADB，並重新啟動 BlueStacks 後再執行環境檢查。

### Bot 一直顯示 `Planning | no actionable target`

這通常不是卡住。Bot 可能在等待安全目標，避免點到我方藍色路徑上的恐龍。
`v0.1.1` 起，即使畫面只辨識到我方藍色路徑，持續多幀後也會自行離開並
重新置中地圖，不會無限掃描同一畫面。

### 在其他裝置登入後，Bot 沒有切回模擬器

確認 BlueStacks 是直向 `900 × 1600`，並讓 Dino Mutant 保持在畫面最上層。新版 Bot
會以多種 UI 尺寸辨識「不同設備的歷史記錄」確認按鈕；若遊戲更新後外觀大幅改變，
仍可能需要更新 `device-history-confirm-button.png` 或 manifest 的 `scales`。

### 重啟後停在「自動成長結果」

新版 Bot 會先開啟「自動戰鬥捷徑」，再點擊快捷視窗外側返回採集地圖。若畫面仍停住，
請保留當時的 `900 × 1600` 截圖與日誌，避免手動改變 UI 後無法重現。

### Bot 點錯或完全辨識不到恐龍

先確認解析度是 `900 × 1600`，並且遊戲 UI 縮放、語言與分享來源相近。不同解析度、DPI 或大幅更新後，可能需要重新製作辨識素材。

### BlueStacks 視窗短暫全黑

先等待約一分鐘。若 ADB 遊戲畫面也持續全黑，Bot 會自動重啟 Dino Mutant App。
`v0.1.1` 起，全黑畫面不會進入目標辨識或點擊驗證，也會明確記錄重啟冷卻剩餘時間。
若每次按下「狩獵」都持續全黑，這是 BlueStacks/Unity 渲染異常，建議完整重啟
BlueStacks，並嘗試切換 BlueStacks 的圖形渲染器。

## 10. 分享給其他人前

請壓縮整個：

```text
D:\DinoMutantBot
```

可先刪除或清空下列資料夾，讓壓縮檔較小且不包含舊日誌：

```text
D:\DinoMutantBot\app\logs
D:\DinoMutantBot\app\debug
D:\DinoMutantBot\app\capture
```

接收者解壓後只需要設定 BlueStacks ADB、修改 `config.json` 的 ADB 路徑，再雙擊
`start-bot.cmd`；啟動器會自行完成環境檢查。
